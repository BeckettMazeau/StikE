# `include/state_types.h`

This header file defines the core data structures and enums that govern the centralized state machine for the StikE UI and event management. It defines what tasks, events, and UI states look like.

## Key Components

### Enums

- **`SystemState`**: Represents the current active screen or mode of the application (e.g., `STATE_UI_LIST`, `STATE_UI_CALENDAR`, `STATE_SLEEP`, `STATE_EPAPER_UPDATE`). State transitions are handled via a FreeRTOS event queue.
- **`CalendarView`**: Represents the active calendar perspective (`MONTH`, `WEEK`, `DAY`).
- **`SystemEventType`**: Defines the type of event generated by the system, typically user input (e.g., `EVENT_NAV_UP`, `EVENT_SELECT`, `EVENT_TYPE_CHAR`, `SLEEP_REQ`).
- **`TaskViewMode`**: Defines the filter for the task list (`ACTIVE`, `COMPLETED`, `BOTH`).
- **`EpaperViewType`**: Defines whether an ePaper view slot holds a `TASK` or an `EVENT`.

### Structs

- **`CalendarEvent`**: Represents a scheduled calendar event.
  - Contains fields for title, notes, location, date (year, month, day, hour, minute), duration, and an optional `linkedTaskId`.
- **`SystemEvent`**: Represents an event in the system's event queue.
  - Contains `type` (the `SystemEventType`) and `param` (an associated integer, often the character code for typing events).
- **`TaskItem`**: Represents a to-do item.
  - Contains fields for title, completion status, creation timestamp, completion date, and optional due date.
- **`EpaperItem`**: A wrapper that can hold either a `TaskItem` or a `CalendarEvent` for rendering on the ePaper display.
- **`EpaperViewItem`**: Represents a single "screen" of information for the ePaper display, containing an array of `EpaperItem`s and a count.

### Constants
- `MAX_TASKS = 20`
- `MAX_CALENDAR_EVENTS = 50`
- `ITEMS_PER_EPAPER_SCREEN = 6`
- `EPAPER_VIEW_COUNT = 10`
- `SLEEP_DURATION_US`: Defines the sleep cycle duration (10 seconds).
- `INPUT_BUFFER_SIZE = 32`

## Dependencies
- `<cstdint>`: Standard integer types.
- `<cstring>`: C-style string manipulation functions (e.g., `strncpy`), used in constructors.
